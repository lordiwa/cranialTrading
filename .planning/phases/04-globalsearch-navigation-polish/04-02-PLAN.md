## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Template attribute binding → DOM | Vue interpolation auto-escapes; aria-* values come from i18n strings (authored) + activeDescendantId (composable-controlled) |
| Keyboard event → handler | KeyboardEvent.key is a browser-controlled enumerated value; IME composition guard prevents bypass |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Information Disclosure | sr-only live region rendered in DOM (visible in HTML source) | accept | Contents are the same visible count already present in the tab labels; no PII, no prices, no user data; same info screen-reader already gets from reading the tabs |
| T-04-02-02 | Tampering | aria-activedescendant binding to composable string | accept | activeDescendantId is set only by moveHighlight/watcher (internal), never by user input; format is `option-{tab}-{index}` with index ≤ 7 |
| T-04-02-03 | Spoofing | role="option" on div.user-result (contains inner button) | accept | Nested interactive elements inside an option are allowed by WAI-ARIA spec; ME INTERESA remains separately focusable via Tab; not a security issue |
| T-04-02-04 | Denial of Service | Arrow key spam triggering moveHighlight rapid-fire | accept | moveHighlight is O(1); no async work; no network calls; Vue reactivity handles re-renders with its own scheduler |
</threat_model>
